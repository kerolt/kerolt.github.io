使用nmcli连接Wi-Fi

现在个人的环境是 Fedora43，使用NetworkManager管理网络，因而基于 nmcli（NetworkManager Command Line Interface）来高效、可靠地配置 WiFi

第一步：确认 NetworkManager 已接管无线接口#

首先确保你的无线网卡（如 wlan0）处于 NetworkManager 的管理之下：

nmcli device status

输出应类似：

DEVICE   TYPE      STATE         CONNECTION 
wlan0    wifi      disconnected  --         
eth0     ethernet  connected     Wired connection 1

🔍 注意：如果 STATE 显示为 unmanaged，说明该设备被排除在 NM 管理之外。检查 /etc/NetworkManager/NetworkManager.conf 是否有 unmanaged-devices 规则，或运行：

sudo nmcli device set wlan0 managed yes

第二步：扫描并连接 WiFi（支持 WPA2/WPA3）#

（1）扫描可用网络（可选，用于确认 SSID 拼写）#

nmcli device wifi list

（2）连接目标 WiFi（自动保存连接配置）#

sudo nmcli device wifi connect "Your_SSID" password "Your_Password"

成功时你会看到：

Device 'wlan0' successfully activated with 'XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX'

💡 底层机制： 此命令会：

  • 调用 wpa_supplicant 后端（通过 D-Bus）
  • 自动生成一个持久化的 connection profile（存储在 /etc/NetworkManager/system-connections/）
  • 自动触发 DHCP 获取 IP（默认行为）
  • 将连接设为 autoconnect=yes，开机自动重连

第三步：验证连接状态与 IP 分配#

# 查看活动连接
nmcli connection show --active

# 查看 wlan0 的 IP
ip addr show wlan0

# 测试连通性
ping -c 3 8.8.8.8

如果无法获取 IP，可能是 DHCP 问题，可手动指定（不推荐长期使用）：

sudo nmcli con modify "Your_SSID" ipv4.addresses 192.168.1.100/24
sudo nmcli con modify "Your_SSID" ipv4.gateway 192.168.1.1
sudo nmcli con modify "Your_SSID" ipv4.dns "8.8.8.8,1.1.1.1"
sudo nmcli con modify "Your_SSID" ipv4.method manual
sudo nmcli con up "Your_SSID"

安全与持久化建议#

  1. 密码加密存储： NetworkManager 默认将密码以加密形式存于 keyring（如 libsecret）或明文存于配置文件（若未启用 keyring）。在无图形界面的服务器上，通常会明文存储于：

    /etc/NetworkManager/system-connections/Your_SSID.nmconnection

    👉 建议设置权限：

    sudo chmod 600 /etc/NetworkManager/system-connections/*.nmconnection
sudo chown root:root /etc/NetworkManager/system-connections/*.nmconnection

  2. 禁用自动连接到开放网络（安全加固）：

    sudo nmcli general set wifi.on-demand on

  3. 查看完整连接配置（调试用）：

    nmcli connection show "Your_SSID"
